# UI / UX Plan

## Main user journeys

### Reader
- Log in
- View approved articles
- View newsletters
- Use API token to retrieve subscribed article feeds

### Journalist
- Log in
- Create article drafts
- Create newsletters from approved articles
- Update or delete own content

### Editor
- Log in
- Open review queue
- Review pending articles
- Approve article
- Confirm article is distributed and logged

## Interface decisions

- A simple navigation bar keeps role-based pages easy to reach.
- Article review has a dedicated queue page for editors.
- Approved article list is separated from the review queue to avoid confusion.
- The interface remains minimal because the capstone focuses on correctness, role logic, and API behaviour rather than advanced front-end styling.
